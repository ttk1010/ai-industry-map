# Claude Design用ブリーフ — AI業界地図スライド

Claude Designに貼り付けて使う想定のブリーフ。ai-industry-mapのデータ
(`data/*.yaml`)を人間が読みやすい形に整理したもの。まずは①AIモデルの
1エリア分（表紙+全体像+モデル一覧+比較表の4枚）で試して、質感が良ければ
残り4エリアに展開する。

## 目的・読み手

- 自分用: 今のAI業界の世界観を1枚で把握する
- 説明用: AIに詳しくない人に「今のAI業界はこんな感じ」を直感的に伝える
- 用途: プレゼン用の**スライド**（Webサイトのようなリンク遷移は不要。
  クオリティ優先）

## トーン・参考にしたい見た目

- `references/ラッコ株式会社_1.png` / `_2.webp` のような、情報密度が高いのに
  整理された、アイコン/マスコットを多用する明るい企業スライド
  （出典: https://zenn.dev/rakko_inc/articles/claude-design-company-deck ）
- 賑やかで親しみやすく、明るい配色。「地味な業界地図」にしたくない
- キャラクターは既存イラストがある場合はそれを踏襲してほしい
  （`references/model-claude-source-crop.png` が実際に生成済みの
  Claudeキャラクター。丸メガネの茶色いふわふわ生物）

## スライド構成案（第一弾: 4枚）

### 1. 表紙
- タイトル: 「AI業界地図」
- サブタイトル: 「AIモデル・チャット・開発環境・ツール・クラウドを1枚で見渡す」

### 2. 全体像（俯瞰）
5つのエリアをタイルまたは地図的なレイアウトで並べる。

| # | エリア名 | 世界観の名前 | 一言 |
|---|---|---|---|
| ① | AIモデル | AI頭脳学園 | 各社の基盤モデルの現在地 |
| ② | チャットアシスタント | AIおしゃべり広場 | 対話インターフェースとしてのAI |
| ③ | 開発環境 | AI開発者タウン | コーディングエージェント／IDEの現在地 |
| ④ | ツール・サービス | MCP港 | いろんな道具・サービスとの連携 |
| ⑤ | クラウド | クラウド王国 | 裏方で支えるインターネットの世界 |

### 3. ①AIモデル 一覧
5社のカードを並べる。各カードの情報:

| モデル | 会社 | ブランドカラー | 一言 | ステータス |
|---|---|---|---|---|
| Claude | Anthropic | #D97757 | 推論と長文コンテキストに強み。Sonnet/Opus/Haikuの三段構成 | 更新あり |
| GPT | OpenAI | #74AA9C | 汎用性とマルチモーダルの王道。ChatGPTとAPIエコシステム | 更新あり |
| Gemini | Google | #3186FF | Google製品群へのネイティブ統合。Computer Use系を強化中 | 更新あり |
| Llama | Meta | #BBA9D6 | オープンウェイトの主流。自前ホスティング・微調整の土台 | 安定 |
| DeepSeek | DeepSeek | #5B7FA6 | 低コスト・高性能を追うオープン系。価格破壊のたび勢力図が揺れる | 要ウォッチ |

ステータスは色で区別（更新あり=暖色、安定=緑、要ウォッチ=赤系）。

Claudeの一言はさらに詳しく: Haiku 4.5(最軽量・低コスト) / Sonnet 5(汎用バランス) /
Opus 4.8(最高性能) / Fable 5(対話・創作寄り、要検証) の4ラインナップがある。

### 4. 比較表: 主要商用LLM（GPT・Claude・Gemini）

出典: `Vault/04 Comparisons/LLM比較- GPT・Claude・Gemini.md`

| 観点 | Claude | GPT | Gemini |
|---|---|---|---|
| 主な強み | 長文処理・要約 | 汎用性・開発者エコシステム | Google連携・マルチモーダル |
| 長文コンテキスト | 非常に大きい | 大きい | 非常に大きい |
| マルチモーダル対応 | 対応あり | 強い | 強い |
| コード支援 | 強い | 強い | 強い |
| エコシステム普及 | 広い | 非常に広い | 広い |
| クラウド/業務連携 | API中心 | Microsoft/Azure中心 | Google Cloud/Workspace中心 |
| 得意な利用場面 | 長文分析・文書作業 | 汎用アプリ開発 | Google基盤の組み込み |
| 注意点 | 地域・提供差がある | モデル差が大きい | 製品名/系統が変わりやすい |
| こんな時に | 長文の読解・要約・文書作業を重視するなら候補 | 汎用API・豊富なSDK・実績重視なら第一候補 | Google Cloud/Workspace統合や画像処理を重視するなら候補 |

## 展開する場合（第二弾以降）の残り4エリアの内容

`data/characters.yaml` と `data/areas.yaml` に全データがある。同じ形式
（カード一覧＋ステータス色分け）で展開できる。

- ③開発環境: Claude Code(Anthropic, #D97757) / Codex(OpenAI, #74AA9C) / Cursor(黒/白, #1A1A1A)
- ②チャットアシスタント: Claude / ChatGPT / Gemini（モデルと同じ色、コーヒーカップがモチーフ）
- ④ツール・サービス: GitHub(#181717) / Notion(#191919) / Slack(#4A154B) / Google Drive(#0F9D58)
- ⑤クラウド: AWS(#FF9900) / Azure(#0078D4) / Google Cloud(#4285F4)
