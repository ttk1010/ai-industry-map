# Claude Design用ブリーフ — AIの世界マップ スライド

Claude Designに貼り付けて使う想定のブリーフ。ai-industry-mapのデータ
(`data/*.yaml`)と、Obsidian Vault（`Vault/01 Concepts/*.md`,
`Vault/04 Comparisons/*.md`）を情報源として整理したもの。

**Claude Code版（`slides/index.html`）とは別物として、ファイルを分けて両方保持する。**
このフォルダ（`claude-design/`）にブリーフと、Claude Designからの出力（PPTX/PDF/
画像など）をまとめる。

## 実装状況（2026-07時点）

Claude Designプロジェクト「Claude企業スライド制作」の `AIの世界マップ.dc.html`
を実装・拡張中（DesignSync経由で取得・配置、および書き戻し）。
- プロジェクト: https://claude.ai/design/p/26f6ed3d-7853-4144-8c85-3f826f72f212
- ローカルで見る: `claude-design/` で `python3 -m http.server 8780` して
  `http://127.0.0.1:8780/AIの世界マップ.dc.html` を開く
- 依存ファイル: `deck-stage.js`（デッキ実行基盤・サムネイルレール等）,
  `support.js`, `assets/*-char.png`（キャラクター）, `assets/logos/*.svg`
- 旧構成（9枚・全5エリア）: 表紙+全体像+①のみフル装備(一覧/比較表/Claude個別)、
  ②③④⑤は一覧のみでした。**この brief.md は以下の「新構成（全30枚）」への
  拡張版です。** 旧構成の名残（表紙の「第一弾」表記、①だけ比較表と個別ページが
  先にある構成など）は新構成で解消する。
- Claude Designで更新したら、DesignSyncで再取得してこのフォルダに反映する
  （list_files → get_file → ローカルへ書き出し）。逆にこちらで編集した時は
  finalize_plan → write_files でプロジェクトへ書き戻す

## 目的・読み手

- 自分用: 今のAI業界の全体像を視覚的に把握する
- 説明用: AIに詳しくない人に「今のAI業界はこんな感じ」を直感的に伝える
- 実務用: 各AI・ツールを実際に使うエンジニアが、選定・導入検討の一次情報として
  参照できる技術的深さを持たせる
- 用途: プレゼン用の**スライド**（Webサイトのようなリンク遷移は不要。
  クオリティ優先）

### 読み手ごとのページの役割分担（重要な設計原則）

このデッキは1種類の読者ではなく、**2種類の読者**を意識してページを作り分ける。

| ページ種別 | 想定読者 | トーン・内容 |
|---|---|---|
| 表紙・全体像 | 誰でも | 一言でわかる世界観、堅苦しくない |
| **エリア概要ページ**（①〜⑤各1枚） | AIに詳しくない人 | そのエリアに何があるか、キャラ・ロゴ・一言紹介のカード一覧。数字や専門用語は最小限 |
| **個別詳細ページ**（各プロダクト1枚ずつ、計18枚） | 実務で使うエンジニア | 料金体系、技術仕様（コンテキスト長・API・SDK）、エコシステム、実務での使いどころ、制約・注意点。専門用語や具体的な仕組みを書いてよい |
| **比較ページ**（①〜⑤各1枚、エリアの最後） | AIに詳しくない人 | どれを選べばいいかが直感的にわかる表。専門用語を避け、「向いている人」で締める |

同じエリアの中で、概要ページと比較ページを混ぜない（これまでの構成の問題点）。
流れは常に **概要 → 個別詳細 ×N → 比較** の順。

## レイアウト方針: 個別詳細ページ（重要 — キャラクター素材の有無で2パターン）

現時点でキャラクターイラストが生成済みなのは **AIモデル4体（Claude/GPT/Gemini/Llama）
のみ**（`assets/*-char.png`）。②チャットの3枚（Claude/ChatGPT/Gemini）は同じ会社・同じ
AIなので、旧構成の「チャット一覧」ページと同様にモデルと同じキャラクターを再利用する。
それ以外（DeepSeek、開発環境3・ツール4・クラウド3の計11プロダクト）はキャラクター未生成
で、ロゴSVGのみが揃っている。そのため個別詳細ページは2つのレイアウトを使い分ける。
将来キャラクターを追加生成したら、内容を変えずにレイアウトAへ差し替え可能な設計にする。

### レイアウトA（キャラクターあり）— Claude/GPT/Gemini/Llama（①）+ Claude/ChatGPT/Gemini（②再利用）の7枚

- 自作キャラクターを**左側に大きく**表示する。スライドの**高さいっぱい**を
  使うイメージ（ただし上部にタイトルがあるので、タイトルの下から高さいっぱい）
- 右側に、会社名・一言・ラインナップ（あれば）・実務ポイント（エンジニア向け）を配置

### レイアウトB（ロゴのみ）— 残り11枚（DeepSeek、開発環境3、ツール4、クラウド3）

- 左側にブランドカラーの円形/角丸バッジの中に、白抜きロゴを大きく配置
  （Claude Codeサイト側の `badge-icon` と同じ考え方。高さいっぱいではなく、
  縦方向中央に大きめのバッジ1つを置くイメージ）
- 右側に、会社名・一言・実務ポイント（エンジニア向け）を配置
- レイアウトAと右側の情報構成は統一し、左側の見た目だけを変える

### 右側の情報構成（共通・両レイアウト）

1. 会社名／提供元
2. 一言（`characters.yaml` の `summary`）
3. （モデル系のみ）ラインナップ・バリエーション
4. 実務ポイント（エンジニア向け）— 3〜5個の短い箇条書き。料金／技術仕様／
   エコシステム／実務での使いどころ／制約のうち、そのプロダクトで特に
   言及価値の高いものを優先して抜粋する（Vaultの実務ポイント全文をそのまま
   貼らず、スライドに収まる分量に要約する）
5. 出典（Vaultノートへのパス）

## 4. 個別詳細: Claude（レイアウトA）

- 会社: Anthropic / ブランドカラー: #D97757
- 一言: 推論と長文コンテキストに強み。Sonnet / Opus / Haiku の三段構成でコスト対応力を確保
- ラインナップ: Haiku 4.5 / Sonnet 5 / Opus 4.8 / Fable 5
- 実務ポイント:
  - 料金はモデル階層ごとにAPI従量課金（上位ほど単価が一桁高い目安）+ Pro/Maxサブスク
  - コンテキストは数十万トークン級。computer use・extended thinking・tool useが特徴機能
  - MCP（Model Context Protocol）の提唱元。Bedrock/Vertex AI経由でも利用可
  - コーディング支援・自律的なエージェントタスクで特に強みを発揮
  - 長文を毎回全量送るとコストが膨らむため、プロンプトキャッシュでの設計が重要
- 出典: `Vault/01 Concepts/Claude.md`

## 5. 個別詳細: GPT（レイアウトA）

- 会社: OpenAI / ブランドカラー: #74AA9C
- 一言: 汎用性とマルチモーダルの王道。ChatGPTとAPIエコシステムの両輪で市場を牽引
- 実務ポイント:
  - 料金はAPI従量課金とChatGPTサブスク（Free〜Enterprise）の二本立て
  - コンテキストは十数万〜百万トークン級。function calling・Structured Outputsが強み
  - 業界最多クラスのサードパーティ連携、Azure OpenAI提供、MCP対応も進行中
  - エコシステムの広さから「まず試す」デファクトの選択肢になりやすい
  - モデル世代交代が速く、旧モデル非推奨（deprecation）への追従コストがかかる
- 出典: `Vault/01 Concepts/GPT.md`

## 6. 個別詳細: Gemini（レイアウトA）

- 会社: Google / ブランドカラー: #3186FF
- 一言: Google製品群へのネイティブ統合。Computer Use系の操作能力を急速に強化中
- 実務ポイント:
  - 料金はAI Studio/Vertex AI経由の従量課金。軽量な「Flash」系はコスト効率が高い
  - コンテキストは最大100万トークン級。学習段階から統合したネイティブマルチモーダルが特徴
  - Google検索と連携した「grounding」で最新情報の引用付き回答が組みやすい
  - Vertex AIでのエンタープライズ運用、Firebase/Android/Workspace統合が強み
  - 無料のAI StudioとエンタープライズのVertex AIで利用条件・レート制限が異なる点に注意
- 出典: `Vault/01 Concepts/Gemini.md`

## 7. 個別詳細: Llama（レイアウトA）

- 会社: Meta / ブランドカラー: #BBA9D6
- 一言: オープンウェイトの主流。自前ホスティングや微調整の土台として定番の地位を維持
- 実務ポイント:
  - モデルの重みは無料。実コストは自前GPUホスティングか第三者推論サービスの従量課金
  - Llama Community Licenseという独自ライセンス。大企業には追加許諾条件がある
  - 数Bクラス〜400B超級まで複数サイズ。量子化でVRAM要件を下げたローカル実行も可能
  - Hugging Face中心のエコシステム、vLLM・Ollama・llama.cppが早期サポート
  - データ主権確保・推論コスト固定化に強み。ただし自己運用のGPUインフラ工数がかかる
- 出典: `Vault/01 Concepts/Llama.md`

## 8. 個別詳細: DeepSeek（レイアウトB — ロゴのみ）

- 会社: DeepSeek / ブランドカラー: #5B7FA6
- 一言: 低コスト・高性能を追う代表的なオープン系モデル。価格破壊のたびに勢力図を揺らす存在
- 実務ポイント:
  - 同等クラスの欧米モデルよりおおよそ一桁安いAPI従量課金。重みもオープンで自己ホスティング可
  - V3系（MoE）と、思考過程を明示するR1系「推論モデル」の二本柱
  - APIはOpenAI互換。既存のOpenAI SDKベースのコードを少ない変更で接続できる
  - コスト重視の大量バッチ処理や、推論性能とコストのバランス重視のコーディング・数学タスクに向く
  - 中国企業が運営するためデータの送信先・保存場所の確認が必要。API可用性も変動しやすい
- 出典: `Vault/01 Concepts/DeepSeek.md`
